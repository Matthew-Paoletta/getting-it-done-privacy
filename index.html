üîê Permissions Justifications

1. activeTab justification

The activeTab permission is required to take a screenshot of the Webreg page when the user clicks the extension icon. This allows the extension to capture the visible class schedule and extract text from the image locally. Access is temporary and only granted after user interaction.

2. storage justification

The storage permission is used to temporarily save the extracted class schedule data and user preferences (like calendar colors) locally in the browser. This lets the user review and edit the schedule before creating the .ics file, with no data sent to external servers.

3. identity justification

The identity permission is only used if the user chooses to directly upload the .ics file to Google Calendar. It enables secure OAuth2 authentication with Google‚Äôs API. No personal identity information is stored or accessed beyond what‚Äôs needed for calendar integration.

4. Host permission justification

No host permissions are required. This extension does not inject scripts or access webpage content. It only uses activeTab to capture a screenshot after user interaction.

5. Remote code usage

No. I am not using remote code. All JavaScript is included in the extension package.
